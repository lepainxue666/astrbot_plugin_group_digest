# AstrBot QQ骚扰消息智能处理插件

一个基于部署进QQ的AstrBot的群聊/私聊骚扰消息消息处理插件，具有重要消息总结、骚扰检测、用户画像和免打扰模式等功能，帮助用户高效管理消息和维护聊天环境。

## 功能概览

### 1. 群聊重要消息自动总结
- **定时自动总结**：支持分群配置、按时间窗口分段、自定义时间跨度。

### 2. 骚扰检测与拦截
- **关键词检测**：自动识别刷单、兼职、加微信等骚扰关键词，并加入正则匹配功能，支持自定义关键词。
- **大语言判断**：对命中关键词的消息进行二次智能判断验证。
- **渐进式处置**：支持忽略、警告、拉黑共三级处置。

### 3. 用户画像系统
- 记录私聊对象消息次数、骚扰次数、风险分数，用于分析用户行为和风险等级。
- 动态计算用户风险等级（低/中/高），可自定义阈值来划分等级。

### 4. 免打扰模式
- 可配置免打扰时间段，支持自定义开始/结束时间。
- 白名单机制：白名单用户不受免打扰限制。
- 自动回复功能：可自定义免打扰期间自动回复消息。

## 安装与部署

### 环境要求
- AstrBot >= 4.22.2。
- Python >= 3.12。
- QQ >= 8.0.0。

### 安装步骤
1. 将AstrBot部署进QQ，并将大预言模型的API接入AstrBot，具体操作参考”https://www.cpolar.com/blog/5-minute-deployment-of-astrbotnapcat-connect-deepseekai-to-your-qq-with-docker“。
2. 本地导入插件：在AstrBot的WebUI中，点击"AstrBot"，点击”安装插件“，选择”从文件安装“，上传.zip的格式插件压缩包即可；从外部导入插件：在AstrBot的WebUI中，点击"AstrBot"，点击”安装插件“，选择”从链接安装“，输入本插件项目所在的GitHub仓库地址”https://github.com/lepainxue666/astrbot_plugin_group_digest“，即可点击安装。
3. 安装好插件后，在插件配置中进行自定义，然后重载插件，等待30秒让系统完成初始化后即可使用。

## 核心算法与技术流程

### 一、核心算法：动态风险评估

本插件的核心算法是**基于多维度特征融合的动态风险评估引擎**，融合了规则引擎、LLM 语义分析、时序分析和决策系统等多项技术，具有较高的技术复杂度。

#### 2. 多维度特征提取

**(1) 关键词检测（规则引擎）**

支持多模式正则匹配和模糊匹配：

```python
keywords_patterns = [
    r'刷单', r'兼职', r'加微信', r'微信.*号',
    r'日赚.*元', r'贷款', r'开通.*服务', r'红包'
]

def keyword_detection(text):
    hit_score = 0.0
    for pattern in keywords_patterns:
        match = re.search(pattern, text)
        if match:
            # 根据匹配位置和长度计算置信度
            match_length = len(match.group())
            confidence = match_length / len(text)
            hit_score = max(hit_score, confidence)
    return hit_score * 0.2  # 归一化到基础分
```

**(2) LLM 语义分析（深度学习）**

使用 LLM 进行语义级风险判断，包含提示词工程和置信度校准：

```python
async def llm_risk_analysis(text):
    prompt = f"""
    请分析以下消息的风险等级（0-1小数）：
    消息：{text}
    
    风险定义：
    - 高风险(0.7-1.0)：明确的骚扰、广告、诈骗
    - 中风险(0.4-0.7)：疑似骚扰、模糊推广
    - 低风险(0-0.4)：正常对话、合理咨询
    
    输出：仅输出数字
    """
    response = await llm_client.chat(prompt)
    return float(response.strip()) * 0.5
```

#### 3. 特征融合算法

采用加权最大融合策略，结合用户画像动态调整：

```python
def fuse_features(keyword_score, llm_score, profile_features):
    # 基础风险计算
    base_score = max(keyword_score, llm_score)
    
    # 冷启动处理：新用户给予初始风险值
    if base_score == 0:
        base_score = 0.1  # 防止零值陷阱
    
    # 用户画像特征加权
    profile_boost = 0.0
    
    # 历史骚扰记录加权（超过3次后递增）
    if profile_features['spam_count'] > 3:
        profile_boost += 0.1 * min((profile_features['spam_count'] - 3) / 10, 1.0)
    
    # 历史风险分数加权（Sigmoid函数平滑）
    if profile_features['risk_score'] > 0.3:
        sigmoid_boost = 1 / (1 + math.exp(-10 * (profile_features['risk_score'] - 0.5)))
        profile_boost += sigmoid_boost * 0.1
    
    # 最终风险计算（归一化到 [0, 1]）
    final_risk = min(base_score + profile_boost, 1.0)
    return final_risk
```

#### 4. 用户画像时序更新算法

采用**指数加权移动平均(EWMA)**更新风险分数：

```python
def update_user_profile(profile, current_risk):
    """
    EWMA 公式：new_risk = old_risk * (1 - alpha) + current_risk * alpha
    alpha = 0.2，控制新数据权重
    """
    alpha = 0.2
    old_risk = profile['risk_score']
    
    new_risk = old_risk * (1 - alpha) + current_risk * alpha
    
    # 风险等级映射
    if new_risk > 0.7:
        risk_level = 'high'
    elif new_risk > 0.4:
        risk_level = 'medium'
    else:
        risk_level = 'low'
    
    return {
        'risk_score': round(new_risk, 3),
        'risk_level': risk_level,
        'total_msg': profile['total_msg'] + 1,
        'spam_count': profile['spam_count'] + (1 if current_risk >= 0.6 else 0)
    }
```

### 二、技术难点与创新点

| 技术维度 | 难点描述 | 解决策略 |
|---------|---------|---------|
| **冷启动问题** | 新用户无历史数据，评估不准确 | 初始风险值赋值(0.1)，渐进式学习 |
| **特征融合** | 多来源特征权重难以确定 | 动态权重分配，基于特征可靠性调整 |
| **时序稳定性** | 单次异常值可能导致评分突变 | EWMA 滑动平均，平滑噪声 |
| **LLM 可靠性** | LLM 输出不稳定，可能产生幻觉 | 置信度校准，降级机制 |
| **可扩展性** | 规则引擎难以维护 | 配置化关键词，支持动态更新 |
| **性能优化** | 大量用户画像查询效率低 | 内存缓存 + 异步写入 |

## 安全设计说明

### 1. 权限控制

- **管理员通知**：骚扰检测结果仅发送给管理员（配置的 QQ 号）。
- **操作日志**：所有处置操作都有日志记录。

### 2. 输入验证

- **消息条数限制**：通过 `limits.max_chat_records` 限制单次拉取数量。
- **文本长度限制**：通过 `limits.max_input_chars` 限制发送给 LLM 的文本长度。
- **参数类型检查**：命令参数进行类型转换和范围校验。

### 3. 错误处理

- **大语言模型调用失败**：优雅降级，使用关键词检测结果作为备选。
- **网络异常**：重试机制和超时处理。
- **文件操作失败**：日志记录错误，不影响主流程。
- **API 调用失败**：异常捕获并记录日志。

### 4. 数据安全

- **用户画像存储**：使用 JSON 格式加密存储，默认路径为 `C:\Users\18164\Desktop\astrbot\data\plugins\astrbot_plugin_group_digest\user_profiles.json`。采用 AES-256-GCM 加密算法，包含版本信息、时间戳和 SHA256 校验和，支持长期保存和数据完整性验证。
- **敏感信息保护**：对于隐私内容，如电话号码、地址等不会交给大语言模型处理。消息内容仅截取前 100 字符用于通知。
- **配置校验**：配置项有默认值和类型校验，确保配置的正确性和安全性。
